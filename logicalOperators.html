Logical Assignment Operators in JavaScript

What are Logical Assignment Operators?
These operators combine logical operations with assignment, making code more concise and readable. They were introduced in ES2021 and are now widely supported in modern browsers.
1. Logical AND Assignment (&&=)

Syntax: x &&= y is equivalent to x && (x = y)

Behavior: Only assigns if the left operand is truthy

let user = { name: "John", email: null }; user.name &&= user.name + " Doe"; // "John Doe" (truthy, so assignment happens) user.email &&= "john@example.com"; // null (falsy, so no assignment)
2. Logical OR Assignment (||=)

Syntax: x ||= y is equivalent to x || (x = y)

Behavior: Only assigns if the left operand is falsy

let user = { name: "", email: null, status: "active" }; user.name ||= "Anonymous"; // "Anonymous" (empty string is falsy) user.email ||= "default@example.com"; // "default@example.com" (null is falsy) user.status ||= "inactive"; // "active" (unchanged, truthy value)
3. Nullish Coalescing Assignment (??=)

Syntax: x ??= y is equivalent to x ?? (x = y)

Behavior: Only assigns if the left operand is null or undefined

let user = { name: "", age: 0, email: null, active: false }; user.name ??= "Default"; // "" (unchanged, not null/undefined) user.age ??= 25; // 0 (unchanged, not null/undefined) user.email ??= "user@example.com"; // "user@example.com" (null triggers assignment) user.active ??= true; // false (unchanged, not null/undefined)
